hike

hike is a small C++11 mathematical optimization library.

It currently only provides a variable neighborhood search (VNS) with first improvement (first descent) and best improvement (highest descent) local search.

Features

  • Header-only library.
  • Solutions can be of any type and size.
  • Loss functions can return any type.
  • Calculated losses can be cached to speedup the optimization process.
  • Low overhead, no heap usage (besides loss caching).
  • Without dependencies (besided catch for testing).

Tested compilers

  • GCC 5.4.
  • MSVC 15.0 (Visual Studio 2017).

Usage

Although hike is a header-only library, a CMakeLists.txt file is provided to facilitate being added as a dependency.

Example

In this catch test, a 3D integer vector is optimized using VNS with first improvement local search:

1 {C++}
2 #include <array>
3 #include <catch.hpp>
4 #include "hikefilocalsearch.h"
5 #include "hikevns.h"
6 
7 TEST_CASE("VNS example")
8 {
9  // Solution is a 3D integer vector. It can be of any type and size:
10  using Solution = std::array<int, 3>;
11 
12  // Loss function return an integer scalar. It can be of any type:
13  struct LossFunction
14  {
15  Solution targetSolution;
16 
17  int getLoss(const Solution& solution) const
18  {
19  int loss = 0;
20 
21  for(std::size_t i = 0; i < solution.size(); ++i)
22  {
23  loss += std::abs(solution[i] - targetSolution[i]);
24  }
25 
26  return loss;
27  }
28  };
29 
30  // Loss function returns the Manhattan distance between the target solution and the given one:
31  Solution targetSolution{{ 2, 5, -10 }};
32  LossFunction lossFunction = { targetSolution };
33 
34  // VNS uses first improvement (first descent) local search.
35  // Best improvement (highest descent) local search can be used too:
36  using LocalSearch = hike::FILocalSearch<Solution, LossFunction>;
37 
38  // Candidate solutions are generated adding and subtracting the parameters of this solution to the given one:
39  Solution stepSolution{{ 1, 1, 1 }};
40 
41  // Declare local search object:
42  LocalSearch localSearch(lossFunction, stepSolution);
43 
44  // Declare VNS object with a maximum neighborhood (kmax) of 5:
45  hike::VNS<Solution, LocalSearch> vns(localSearch, 5);
46 
47  // Optimize a solution:
48  Solution solution{{ 15, -7, 22 }};
49  bool optimized;
50  Solution optimizedSolution = vns.optimize(solution, optimized);
51 
52  // The optimized solution should be equals to the target one:
53  REQUIRE(optimized);
54  REQUIRE(optimizedSolution == targetSolution);
55 }

Using loss caching

With hike::CachedLossFunction class, calculated losses for the same solutions are cached to avoid to recalculate them in next iterations:

1 {C++}
2 #include <array>
3 #include <catch.hpp>
4 #include "hikecachedlossfunction.h"
5 #include "hikefilocalsearch.h"
6 #include "hikevns.h"
7 
8 // Solution is a 3D integer vector. It can be of any type and size:
9 using Solution = std::array<int, 3>;
10 
11 // CachedLossFunction uses std::hash to identify a solution, so it must be overloaded for the solution type:
12 namespace std
13 {
14  template<>
15  struct hash<Solution>
16  {
17  std::size_t operator()(const Solution& solution) const
18  {
19  std::size_t hash = 0;
20 
21  for(int param : solution)
22  {
23  // https://stackoverflow.com/questions/2590677/how-do-i-combine-hash-values-in-c0x:
24  hash ^= std::hash<int>()(param) + 0x9e3779b9 + (hash << 6) + (hash >> 2);
25  }
26 
27  return hash;
28  }
29  };
30 }
31

33 {
34  // Loss function return an integer scalar. It can be of any type:
35  struct LossFunction
36  {
37  Solution targetSolution;
38 
39  int getLoss(const Solution& solution) const
40  {
41  int loss = 0;
42 
43  for(std::size_t i = 0; i < solution.size(); ++i)
44  {
45  loss += std::abs(solution[i] - targetSolution[i]);
46  }
47 
48  return loss;
49  }
50  };
51 
52  // Loss function returns the Manhattan distance between the target solution and the given one:
53  Solution targetSolution{{ 2, 5, -10 }};
54  LossFunction lossFunction{targetSolution};
55 
56  // Declare cached loss function object using lossFunction as implementation:
57  using CachedLossFunction = hike::CachedLossFunction<Solution, LossFunction, int>;
58  CachedLossFunction cachedLossFunction(lossFunction);
59 
60  // VNS uses first improvement (first descent) local search.
61  // Best improvement (highest descent) local search can be used too:
62  using LocalSearch = hike::FILocalSearch<Solution, CachedLossFunction>;
63
